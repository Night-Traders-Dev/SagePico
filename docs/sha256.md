# SHA-256 Hardware Accelerator

RP2350 hardware SHA-256 hashing engine. Computes SHA-256 digests in hardware with minimal CPU overhead.

## Usage

```
>>> sha256("hello")
2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824
```

### Pure Sage API

```sage
import sha256
let digest = sha256.hash("hello world")
```

### FFI (on-device)

```python
ffi_call(pico, "sha256", ["hello world"])
# → "b94d27b9934d3e08a52e52d7da7dabfac484efe37a5380ee9088f7ace2efcde9"
```

## Hardware Details

- **Engine**: RP2350 SHA-256 hardware accelerator
- **Block size**: 64 bytes (512 bits)
- **Digest size**: 32 bytes (256 bits) → 64 hex characters
- **Endianness**: Little-endian output (matches standard SHA-256)

## C Bridge API

```c
const char* sage_sha256_hash(const uint8_t* data, uint32_t len);
const char* sage_sha256_str(const char* s);
```

Returns pointer to static 65-byte buffer (64 hex chars + null). Valid until next call.

## Implementation

The hardware accelerator is fed byte-by-byte via `sha256_put_byte()`. For bulk data, `sha256_put_word()` (32-bit) or DMA (`sha256_set_dma_size()`) offer higher throughput.

Current implementation uses byte-at-a-time for simplicity. For large data (>1KB), consider the DMA path for performance.

## Known Limitations

- Byte-at-a-time feeding limits throughput (~1-2 MB/s)
- Static output buffer (not thread-safe)
- No streaming API (must hash complete data at once)
- SHA-256 only (no SHA-1, SHA-512 — RP2350 only has SHA-256)
